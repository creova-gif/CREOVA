ULTIMATE WEBSITE BRUTE TEST PROMPT
Writing

You are a world-class QA engineer, product manager, UX designer, accessibility expert, and growth strategist.

Your task is to perform a COMPLETE, BRUTAL, END-TO-END audit of my website.

Do NOT be polite. Be extremely critical, detailed, and structured. Identify everything broken, confusing, slow, or suboptimal. Think like a first-time user, a paying customer, and a malicious tester.

Website URL: [INSERT LINK]

1. FIRST IMPRESSION (0–10 seconds)
What is the immediate understanding of the product?
Is the value proposition clear within 3 seconds?
What emotions does the design evoke?
What is confusing, cluttered, or missing?
Would a user trust this site instantly? Why or why not?
2. USER FLOWS (FULL JOURNEY TEST)
2.1 New User Flow
Landing → Browse → Sign Up → Onboarding → First Action
Identify friction points
Count clicks to value
Any unclear steps?
Drop-off risks?
2.2 Authentication
Sign up flow (email, Google, etc.)
Login flow
Password reset flow
Error handling quality
Security perception
2.3 Core Product Flow

Test ALL major actions:

Creating an account
Adding content/products/properties
Editing & deleting
Uploading images/files
Searching/filtering
Checkout/payment (if applicable)

For each:

Is it intuitive?
Is feedback clear?
Any delays or bugs?
Is success/failure communicated properly?
2.4 Payment Flow (CRITICAL)
Add to cart → Checkout → Payment → Confirmation
Trust signals (SSL, branding, UI)
Any hesitation points?
Form friction?
Failed payment handling?
Refund clarity?
3. UI / UX DESIGN AUDIT
3.1 Layout & Visual Hierarchy
Is attention guided correctly?
Are important elements obvious?
Is anything overwhelming or hidden?
3.2 Colors
Are colors consistent?
Do they match brand identity?
Contrast issues (readability)
Emotional tone (trust, premium, playful, etc.)
3.3 Typography
Font readability
Hierarchy (headings vs body)
Consistency
3.4 Buttons & CTAs
Are CTAs obvious?
Are they compelling?
Are there too many or too few?
3.5 Animations & Microinteractions
Are animations smooth or laggy?
Do they add value or distract?
Any missing feedback (hover, click, loading)?
4. PERFORMANCE & SPEED
Page load time (per page)
Image optimization
Lazy loading
Any lag or freezing?
Mobile performance vs desktop
5. RESPONSIVENESS (CRITICAL)

Test on:

Laptop (desktop browser)
Tablet (iPad)
Mobile phone

For EACH device:

Layout breaks?
Elements overlapping?

Buttons hard to tap?
Navigation usability?
